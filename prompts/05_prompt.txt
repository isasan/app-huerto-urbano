Ahora implementa el Historial de Cosechas y los endpoints del Dashboard.

## TAREA BACKEND

### HarvestService.java y HarvestController.java
- CRUD bajo /api/harvests
- Filtrar siempre por userId del token
- GET    /api/harvests              → todas las cosechas del usuario
- GET    /api/harvests?cropId=X    → cosechas de un cultivo específico
- POST   /api/harvests
- PUT    /api/harvests/{id}
- DELETE /api/harvests/{id}
- HarvestRequest: harvestDate, quantity, unit, quality, notes, cropId
- HarvestResponse: todos los campos + plantName + gardenName

### DashboardController.java → GET /api/dashboard

Devuelve un DashboardResponse con:
- totalGardens: número de huertos del usuario
- totalCrops: cultivos activos (no HARVESTED)
- pendingTasks: tareas pendientes
- overdueTasks: tareas vencidas (dueDate < hoy y no completadas)
- recentHarvests: últimas 5 cosechas
- cropsReadyToHarvest: cultivos con status READY
- tasksDueToday: tareas con dueDate = hoy
