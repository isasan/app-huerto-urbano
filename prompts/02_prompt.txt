Continuamos con el proyecto HuertoApp (Spring Boot 3 + Java 21 + Vue 3).
El scaffolding ya está creado. Ahora implementa el módulo de autenticación completo.

## TAREA BACKEND

Implementa completamente estos archivos:

### JwtTokenProvider.java
- Generar token JWT firmado con HS256
- Validar token y capturar excepciones (expirado, malformado)
- Extraer username y rol del token
- Usar la clave y expiración de application.properties

### JwtAuthenticationFilter.java
- Extender OncePerRequestFilter
- Leer header Authorization: Bearer {token}
- Validar token y cargar UserDetails en el SecurityContext

### UserDetailsServiceImpl.java
- Implementar UserDetailsService
- Cargar usuario por username desde UserRepository
- Mapear rol a GrantedAuthority

### SecurityConfig.java
- CSRF deshabilitado (API REST)
- CORS habilitado para http://localhost:5173 (Vite dev server)
- Rutas públicas: POST /api/auth/**, GET /api/calendar/**, GET /api/weather/**
- Resto de rutas requieren autenticación
- Añadir JwtAuthenticationFilter antes de UsernamePasswordAuthenticationFilter

### AuthService.java
- register(): validar que username y email no existen, hashear password con BCrypt, guardar usuario con rol USER por defecto
- login(): autenticar credenciales, generar y devolver JWT
- Lanzar excepciones descriptivas en caso de error

### AuthController.java
- POST /api/auth/register → RegisterRequest → AuthResponse
- POST /api/auth/login → LoginRequest → AuthResponse
- AuthResponse incluye: token, username, role, expiresIn

## TAREA FRONTEND

### stores/auth.js (Pinia)
- Estado: token, user (username + role), isAuthenticated
- Actions: login(), register(), logout()
- Persistir token en localStorage
- Restaurar sesión al recargar la página

### services/authService.js
- login(credentials): POST /api/auth/login
- register(userData): POST /api/auth/register

### services/api.js
- Instancia Axios con baseURL: http://localhost:8080/api
- Interceptor request: añadir Authorization: Bearer {token} si existe
- Interceptor response: si 401 → logout y redirigir a /login

### router/index.js — Navigation Guards
- Rutas protegidas: redirigir a /login si no autenticado
- Ruta /admin: redirigir a /dashboard si no es ADMIN
- Si ya autenticado y va a /login o /register: redirigir a /dashboard

### LoginView.vue
- Formulario: username + password
- Validación básica de campos vacíos
- Mostrar error del servidor si credenciales incorrectas
- Redirigir a /dashboard tras login exitoso
- Link a /register

### RegisterView.vue
- Formulario: username, email, password, confirmPassword, ciudad, hemisferio
- Validar que passwords coinciden
- Mostrar errores de validación
- Redirigir a /login tras registro exitoso

- Loading states en los botones de submit
- No guardar password en el store de Pinia, solo el token
