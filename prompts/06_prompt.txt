Continuamos con HuertoApp. Ahora implementa la integración con OpenWeatherMap.

## TAREA BACKEND

### WeatherService.java
- Llamar a OpenWeatherMap API: GET https://api.openweathermap.org/data/2.5/weather
- Parámetros: q={city}, appid={apiKey}, units=metric, lang=es
- Implementar caché en memoria de 30 minutos por ciudad (Map<String, CachedWeather>)
- CachedWeather: datos + timestamp, invalidar si han pasado 30 min
- Devolver WeatherResponse con: ciudad, temperatura, sensación térmica,
  descripción, icono, humedad, velocidad viento, visibilidad
- Manejar ciudad no encontrada (404 de OpenWeatherMap → error descriptivo)

### WeatherController.java
- GET /api/weather?city={city} → WeatherResponse
- Si city está vacío → usar la ciudad del usuario autenticado (del token)
- Si el usuario no tiene ciudad configurada → error descriptivo

### UserController.java (nuevo o añadir a AuthController)
- GET  /api/users/me           → perfil del usuario autenticado
- PUT  /api/users/me           → actualizar ciudad y hemisferio
- PUT  /api/users/me/password  → cambiar contraseña (requiere password actual)

## CONSIDERACIONES
- La API key de OpenWeatherMap debe estar SOLO en el backend, nunca expuesta al frontend
- El free tier tiene límite de 60 llamadas/minuto: la caché de 30min es suficiente
- Iconos disponibles en: https://openweathermap.org/weather-conditions
