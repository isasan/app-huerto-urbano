Eres un desarrollador fullstack experto en Spring Boot y Vue 3.
Vamos a construir una aplicación de gestión de huertos urbanos domésticos llamada "HuertoApp".

## CONTEXTO DEL PROYECTO
- MVP para portafolio personal fullstack
- Usuarios gestionan sus propios huertos caseros
- Página principal con calendario de siembra estático por hemisferio
- Widget de clima por ciudad del usuario
- Todo en español

## STACK TECNOLÓGICO

### Backend
- Java 21
- Spring Boot 3.x (última estable)
- Spring Security 6 + JWT (jjwt 0.12.x)
- Spring Data JPA + Hibernate
- Maven
- H2 Database en modo fichero (no en memoria) para persistencia local
- Lombok para reducir boilerplate
- OpenWeatherMap API para el clima (free tier)

### Frontend
- Vue 3 (Composition API, <script setup>)
- Vite
- Bootstrap 5
- Pinia (state management)
- Vue Router 4
- Axios

## MODELOS DE DATOS

### User
- id (Long, PK)
- username (String, unique)
- email (String, unique)
- password (String, bcrypt)
- role (Enum: ADMIN, USER)
- city (String)
- hemisphere (Enum: NORTE, SUR)
- createdAt (LocalDateTime)

### Garden
- id (Long, PK)
- name (String)
- description (String)
- location (String)
- sizeM2 (Double)
- user (ManyToOne → User)
- createdAt (LocalDateTime)

### Plot
- id (Long, PK)
- name (String)
- sizeM2 (Double)
- soilType (String)
- garden (ManyToOne → Garden)

### Crop
- id (Long, PK)
- plantName (String)
- variety (String)
- plantedDate (LocalDate)
- expectedHarvestDate (LocalDate)
- status (Enum: PLANTED, GROWING, READY, HARVESTED)
- notes (String)
- plot (ManyToOne → Plot)

### Task
- id (Long, PK)
- title (String)
- type (Enum: RIEGO, FERTILIZAR, PODAR, TRATAR, OTRO)
- dueDate (LocalDate)
- completed (Boolean)
- notes (String)
- crop (ManyToOne → Crop, nullable)
- garden (ManyToOne → Garden)

### HarvestLog
- id (Long, PK)
- harvestDate (LocalDate)
- quantity (Double)
- unit (String)
- quality (Enum: MALA, NORMAL, BUENA, EXCELENTE)
- notes (String)
- crop (ManyToOne → Crop)

## DATOS ESTÁTICOS — CALENDARIO DE SIEMBRA

Crea SeedingCalendarData.java con estas plantas para cada hemisferio:
tomate, lechuga, zanahoria, pimiento, pepino, calabacín, espinacas, cebolla, ajo, perejil, albahaca, fresas.

Cada planta incluye: nombre, emoji, meses de siembra, meses de trasplante,
meses de cosecha, dificultad (FACIL/MEDIA/DIFICIL), días hasta cosecha.

## RUTAS FRONTEND

/                → HomeView (pública)
/login           → LoginView (pública)
/register        → RegisterView (pública)
/dashboard       → DashboardView (auth requerida)
/gardens         → GardensView (auth requerida)
/gardens/:id     → GardenDetailView (auth requerida)
/tasks           → TasksView (auth requerida)
/harvests        → HarvestView (auth requerida)
/admin           → AdminView (solo ADMIN)

## SEGURIDAD

- Rutas públicas: GET /api/calendar, GET /api/weather, POST /api/auth/**
- Rutas protegidas: JWT válido en header Authorization: Bearer {token}
- Cada usuario solo accede a sus propios recursos (validar en Service layer)
- Rol ADMIN puede ver todos los usuarios y gestionar catálogo de plantas

## TAREA

Genera el scaffolding completo del proyecto:

1. Crea el pom.xml con todas las dependencias necesarias
2. Crea todos los archivos Java con anotaciones, imports y métodos con TODO
3. Crea el application.properties
4. Crea los modelos con todas sus anotaciones JPA y Lombok
5. Crea el package.json del frontend y vite.config.js con proxy al puerto 8080
6. Crea todos los archivos Vue con estructura <script setup> + <template> + <style>
7. Crea el router/index.js con rutas y navigation guards
8. Crea api.js con interceptor JWT y manejo de error 401
9. Crea SeedingCalendarData.java con datos completos de las 12 plantas para ambos hemisferios
10. Al finalizar muestra un resumen de todos los archivos creados agrupados por capa

Empieza por el backend completo y luego el frontend. No omitas ningún archivo.
